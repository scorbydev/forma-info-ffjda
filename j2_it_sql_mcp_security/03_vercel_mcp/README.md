# Atelier 3 - Vercel MCP

Objectif: voir Codex connecte a un service externe via MCP, en distinguant clairement les droits de lecture, ecriture et suppression.

Vercel est retenu car un petit projet frontend peut etre construit et deploye rapidement avec une offre gratuite adaptee a une demonstration. Ce choix est coherent avec l'interet des participants pour les applications generees ou prototypees avec des outils type Lovable. La mini app est volontairement simple, sans backend, sans base et sans donnee reelle.

Le succes pedagogique ne depend pas du MCP:

- si OAuth fonctionne, inspecter projets, deploiements et logs via MCP;
- si OAuth bloque, utiliser Vercel CLI et `fallback_without_mcp.md`;
- si aucun compte n'est disponible, construire localement et expliquer les actions qu'un MCP aurait automatisees.

Commencer par `vercel_mcp_playbook.md`.
